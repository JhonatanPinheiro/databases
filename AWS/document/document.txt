Seções do módulo:

•Amazon Relational Database Service (Amazon RDS)
•Amazon DynamoDB•Amazon Redshift
•Amazon Aurora


- Amazon RDS (Relational Database Service) é um serviço de banco de dados gerenciado pela AWS. Ele simplifica tarefas como configuração, operação e escalabilidade de bancos de dados na nuvem. A principal diferença entre um serviço gerenciado, como o RDS, e um não gerenciado é que, no gerenciado, a AWS cuida da infraestrutura, backups, atualizações e alta disponibilidade, enquanto no modelo não gerenciado, todas essas responsabilidades ficam por conta do usuário. O Amazon RDS também permite configurar ambientes altamente disponíveis com réplicas em múltiplas zonas de disponibilidade (Multi-AZ), garantindo resiliência e continuidade dos serviços.

- Amazon DynamoDB é um serviço de banco de dados NoSQL totalmente gerenciado, projetado para oferecer desempenho rápido e consistente em qualquer escala. Ele utiliza particionamento automático de dados para distribuir e equilibrar a carga entre múltiplos nós, permitindo lidar com grandes volumes de dados e tráfego elevado. Essa arquitetura permite que o DynamoDB aumente a escala horizontal sob demanda, sem interrupções, tornando-o ideal para aplicações que exigem alta disponibilidade e baixa latência.

- Amazon Redshift é um serviço de data warehouse totalmente gerenciado, projetado para análise de grandes volumes de dados. Ele utiliza uma arquitetura de processamento massivamente paralelo (MPP), que divide consultas em partes menores e as executa simultaneamente em vários nós. Isso permite processar rapidamente conjuntos de dados extensos e complexos. É amplamente usado para BI (Business Intelligence), análise em tempo quase real, integração com ferramentas como Tableau e integração de dados de múltiplas fontes para geração de relatórios e insights estratégicos.

- Amazon Aurora é um banco de dados relacional compatível com MySQL e PostgreSQL, projetado para oferecer desempenho superior e maior resiliência em comparação ao Amazon RDS tradicional. É ideal para aplicações que exigem alta performance, disponibilidade e escalabilidade. Aurora se destaca em cenários com grande volume de leitura e escrita, e oferece replicação automática entre múltiplas zonas de disponibilidade (Multi-AZ), garantindo alta disponibilidade e recuperação rápida em caso de falhas.
